================================================================================
ENVIRONMENT VARIABLES REQUIRED FOR RELEASE
================================================================================

RELEASE_KEYSTORE_PATH     - Path to your production keystore file
RELEASE_KEYSTORE_PASSWORD - Keystore password
RELEASE_KEY_ALIAS         - Key alias (e.g., "focuslock")
RELEASE_KEY_PASSWORD      - Key password

Example (Linux/Mac):
export RELEASE_KEYSTORE_PATH="/path/to/focuslock-release.jks"
export RELEASE_KEYSTORE_PASSWORD="your_password"
export RELEASE_KEY_ALIAS="focuslock"
export RELEASE_KEY_PASSWORD="your_key_password"

================================================================================
SECURITY NOTES
================================================================================

CRITICAL: Your production keystore is the ONLY way to update your app on 
Play Store. If lost, you cannot update your app and must create a new listing 
(losing all reviews, ratings, and downloads).

Actions Required:
1. Store keystore file in multiple secure locations
2. Never commit keystore to version control
3. Use strong passwords (12+ characters minimum)
4. Keep encrypted backups
5. Document keystore location for team
